# ClickSyncMouseTester 使用说明书

ClickSyncMouseTester 是一款面向鼠标输入测试的 Windows 工具，用于观察鼠标在实际软件环境中的 Raw Input 上报表现。

本说明书会按功能逐步介绍软件的用途、操作方法、指标含义和基本原理。

---

## 1. 回报率检测

### 1.1 这个功能是做什么的

回报率检测用于观察鼠标向 Windows 上报输入数据的频率。

常见鼠标标称回报率包括：

- 125 Hz
- 500 Hz
- 1000 Hz
- 2000 Hz
- 4000 Hz
- 8000 Hz

回报率越高，理论上鼠标位置更新越频繁，输入延迟越低。但实际表现会受到鼠标固件、接收器、USB、系统调度、驱动和当前负载影响。

本功能检测的是：

> Windows Raw Input 层实际收到的鼠标报告速率。

它不是 USB 总线物理层测试仪，因此结果更适合用来判断鼠标在当前电脑和系统环境下的实际输入表现。

---

### 1.2 什么时候需要使用

你可以用它检查：

- 鼠标是否接近标称回报率。
- 8000 Hz / 4000 Hz 鼠标是否真的在高频上报。
- 无线模式下回报是否稳定。
- 不同 USB 接口、接收器位置、驱动设置是否影响回报率。
- 鼠标移动时是否存在明显丢包、波动或空包。

---

### 1.3 基本使用步骤

1. 打开软件，进入 **回报率检测** 页面。
2. 在右侧选择需要测试的鼠标设备。
3. 选择统计模式：
   - **Raw Packet**
   - **Motion Report**
4. 点击开始测试。
5. 在采集区域内持续快速画圈 **3-5 秒**。
6. 观察主数值和底部指标。
7. 按 `Esc` 或右键退出锁定状态。

测试时建议：

- 鼠标持续移动，不要只轻微移动。
- 尽量快速、连续地画圈。
- 测试 3-5 秒，比只滑动一下更可靠。
- 高回报率鼠标建议关闭明显占用 CPU/GPU 的后台程序。

---

### 1.4 两种统计模式

#### Raw Packet

Raw Packet 会统计所选鼠标发出的全部 Raw Input 鼠标包。

包括：

- 有位移的运动包
- 零位移包
- 按键相关包
- 滚轮相关包

适合观察鼠标对 Windows 的整体上报频率。

如果你只想看“系统实际收到多少鼠标报告”，优先使用这个模式。

#### Motion Report

Motion Report 只统计有运动的报告：

```text
DeltaX != 0 或 DeltaY != 0
```

它会过滤零运动包，更接近“鼠标实际移动期间的位置更新频率”。

适合观察：

- 移动期间的有效回报率
- 零运动包占比
- 高回报率鼠标移动时是否稳定

---

### 1.5 主数值：Raw Input Report Rate

页面中央的大数字是当前 Raw Input Report Rate。

它表示最近约 300ms 短时间窗口内，软件从 Windows Raw Input 收到的鼠标报告速率，单位是 Hz。

例如：

```text
1000 Hz
```

表示当前约每秒收到 1000 个鼠标报告。

注意：

- 这个数值是实时估算值，会正常上下波动。
- 高回报率鼠标需要持续快速移动才能测准。
- 低速移动可能低估高回报率设备。

---

### 1.6 底部指标说明

#### 峰值速率

峰值速率表示本次测试中曾经达到过的较高回报率。

它用于快速判断鼠标是否曾接近标称值。

例如：

- 标称 1000 Hz，峰值接近 1000 Hz：通常正常。
- 标称 8000 Hz，峰值长期只有 1000 Hz：可能设置、接口、接收器或系统环境存在限制。

#### 1 秒间隔抖动

1 秒间隔抖动表示最近 1 秒内，鼠标报告之间的时间间隔稳定程度。

它是百分比指标：

```text
间隔抖动 = 报告间隔标准差 / 平均报告间隔 × 100%
```

简单理解：

- 数值越低，报告间隔越稳定。
- 数值越高，报告间隔越不均匀。

这个指标不是回报率本身，而是回报稳定性的辅助指标。

为了避免刚开始测试时数据不稳定，软件会在检测到第一个有效包后等待约 1 秒，再显示该指标。

#### 空包率

空包率只在 **Motion Report** 模式显示。

它表示最近 1 秒内，Raw Input 包中零运动包的占比：

```text
DeltaX == 0 且 DeltaY == 0
```

空包率用于观察鼠标在移动测试期间，有多少报告没有实际位移。

注意：

- 空包率高不一定代表鼠标有问题。
- 它需要结合测试方式、DPI、移动速度和鼠标固件策略判断。
- 软件会从第一个有效运动包后开始统计，避免把“开始测试后还没移动”的空闲时间算进去。

#### 丢包计数

丢包计数表示软件内部缓冲队列出现溢出时丢弃的包数量。

正常情况下应为 0。

如果持续增加，通常说明：

- 系统负载过高。
- 回报率过高导致处理压力变大。
- 当前环境无法稳定消费所有 Raw Input 包。

---

### 1.7 暂停和继续

暂停时，软件不会重新计算暂停瞬间的数据，而是冻结最近一次正常刷新得到的快照。

这样可以避免：

- 点击停止或右键退出带来的异常包。
- 停止瞬间的长间隔污染抖动结果。
- 暂停时指标突然跳到异常值。

继续测试后，1 秒间隔抖动和 1 秒空包率会重新等待有效包并完成 1 秒 warm-up 后再显示。

---

### 1.8 如何理解测试结果

建议重点看三件事：

1. **主数值是否接近标称回报率**
2. **1 秒间隔抖动是否明显偏高**
3. **丢包计数是否保持为 0**

如果回报率偏低，可以尝试：

- 检查鼠标驱动或控制软件中的回报率设置。
- 更换 USB 接口。
- 无线鼠标靠近接收器。
- 关闭省电模式。
- 降低后台负载。
- 使用 Motion Report 模式重新测试。

---

### 1.9 原理简述

Windows 会通过 Raw Input 向应用程序提供底层输入数据。

ClickSyncMouseTester 监听指定鼠标设备的 Raw Input 包，并记录每个包到达的时间戳。

软件根据这些时间戳计算：

- 最近约 300ms 短窗口内的当前回报率。
- 最近 1 秒内的报告间隔稳定性。
- 最近 1 秒内零运动包占比。
- 内部缓冲是否发生丢包。

因此，它反映的是：

> 鼠标在当前 Windows 软件层环境中的实际上报表现。

这也是普通软件工具在 Windows 用户态下最接近真实使用体验的检测方式之一。

## 2. 按键双击检测

### 2.1 这个功能是做什么的

按键双击检测用于观察鼠标按键、滚轮和自定义键位的输入状态与时序。

它可以检测：

- 左键、中键、右键、前进键、后退键的按下和抬起。
- 鼠标滚轮上滚、下滚次数。
- 按下到下一次按下的间隔。
- 按下到抬起的持续时间。
- 在指定阈值内发生的双击次数。
- 用户录入的键盘按键或特殊键位。

这个页面适合排查按键连点、误双击、回弹不稳定、微动老化、滚轮误触等问题。

---

### 2.2 基本使用步骤

1. 进入 **按键双击检测** 页面。
2. 直接点击或按下需要测试的鼠标按键。
3. 观察对应卡片上的按下计数、抬起计数、双击次数和时间指标。
4. 转动滚轮，观察上滚和下滚计数。
5. 如需检测键盘或其他键位，使用 **自定义键位检测** 区域。
6. 点击 **重置统计** 清空本页所有统计结果。

页面会实时高亮当前处于按下状态的按键。如果按键释放后仍显示按下，通常说明系统没有收到对应抬起事件，或者鼠标硬件/驱动上报存在异常。

---

### 2.3 鼠标双击阈值

页面中的 **鼠标双击阈值** 用于判断两次“按下”之间是否算作双击。

例如阈值设置为：

```text
80 ms
```

如果同一个按键两次按下之间的间隔小于 80ms，软件会把它计入双击次数。

建议范围通常为 **60-120ms**：

- 阈值越小，判断越严格，更适合排查极短连点。
- 阈值越大，越容易把连续点击判为双击。

如果你想模拟游戏或系统中更严格的双击判定，可以调低阈值；如果你想观察微动是否容易短时间内重复触发，可以适当调高阈值。

---

### 2.4 指标说明

#### 双击次数

双击次数表示同一按键两次按下之间的时间小于当前双击阈值的次数。

如果你只按了一次，却出现双击次数增加，通常说明按键存在异常回弹或重复上报。

#### 按下计数 / 抬起计数

按下计数表示软件收到的 Down 事件数量。

抬起计数表示软件收到的 Up 事件数量。

正常点击时，两者通常应接近一致。如果按下计数明显多于抬起计数，可能存在重复按下上报；如果抬起计数异常，可能存在释放事件丢失或设备状态不同步。

#### 按下→按下

按下→按下表示本次按下与上一次按下之间的间隔。

页面会显示当前值、最小值和平均值。这个指标适合观察：

- 连点时每次触发间隔是否稳定。
- 微动是否在极短时间内重复触发。
- 手动双击速度大概是多少。

#### 按下→抬起

按下→抬起表示一次按住持续了多久。

它可以帮助判断：

- 一次点击的按住时长。
- 长按是否被稳定保持。
- 是否存在刚按下就异常抬起的情况。

---

### 2.5 滚轮检测

滚轮区域会分别统计：

- **上滚**
- **下滚**

每次收到滚轮上报后，对应方向会增加计数并短暂高亮。

如果滚轮实际向上滚却出现下滚计数增加，或者一次滚动出现大量反方向计数，可能说明编码器老化、滚轮结构松动或驱动层存在异常。

---

### 2.6 自定义键位检测

自定义键位检测用于检测不在固定鼠标按键卡片中的键位。

使用方法：

1. 点击 **录入**。
2. 按一次需要检测的键。
3. 页面显示已选择的键位后，开始正常按下和抬起该键。
4. 观察按下计数、抬起计数、双击次数和时间指标。
5. 点击 **重置** 清空当前自定义键位统计。
6. 再次点击 **录入** 可以重新选择键位。

录入状态下可按 `Esc` 取消。

自定义键位也有独立的双击阈值，仅影响自定义键位的双击判断，不影响鼠标固定按键区域。

---

### 2.7 如何理解测试结果

建议重点看四件事：

1. **按一次是否只增加一次按下和一次抬起**
2. **双击次数是否在不该出现时增加**
3. **按下→按下的最小值是否异常接近 0**
4. **按下→抬起是否符合你的实际按住时长**

如果怀疑鼠标双击，可以这样测试：

- 把鼠标双击阈值设置到 60-80ms。
- 用正常单击力度连续点击同一按键。
- 观察双击次数是否异常增加。
- 换不同力度、不同位置点击，确认是否稳定复现。

如果只有极偶发的双击，不一定能在几次点击中测出；建议连续测试几十次，并结合按下→按下最小值判断。

---

### 2.8 原理简述

ClickSyncMouseTester 通过 Raw Input 监听鼠标按键、滚轮和键盘输入事件。

每次收到按下事件时，软件记录时间戳，并与上一次按下时间比较，得到按下→按下间隔。

每次收到抬起事件时，软件用抬起时间减去本次按下时间，得到按下→抬起时长。

双击判断只基于两次按下事件之间的时间是否小于当前阈值，不会修改系统双击速度设置，也不会改变鼠标硬件参数。

## 3. 鼠标性能分析

### 3.1 这个功能是做什么的

鼠标性能分析用于锁定采集某一只鼠标的 Raw Input 数据，并在独立图表窗口中观察更细的运动、时序和分布表现。

它比回报率检测更适合做长一点的会话分析，例如：

- 查看某次移动中的每包 DeltaX / DeltaY。
- 观察上报间隔是否有尖峰。
- 对比不同鼠标或不同设置下的轨迹和速度。
- 导出采集数据，之后再导入复查或对比。
- 保存图表 PNG，方便记录测试结果。

---

### 3.2 基本使用步骤

1. 进入 **鼠标性能分析** 页面。
2. 在 **目标设备** 中选择要分析的鼠标。
3. 在 **鼠标DPI** 中输入当前鼠标 CPI / DPI。
4. 点击 **开始采集**，或按 `S` 进入锁定采集。
5. 在锁定状态下移动鼠标，完成你想测试的动作。
6. 再次按 `S`、按 `Esc`、右键，或切出窗口，结束本次采集段。
7. 点击 **图表** 打开独立图表窗口。
8. 如需继续追加数据，点击 **继续采集** 或再次按 `S`。
9. 点击 **重置** 清空当前会话。

采集段结束后，数据会保留在当前会话中。继续采集会追加到同一会话，只有重置才会清空。

---

### 3.3 DPI / CPI 输入

DPI / CPI 用于把鼠标计数换算为物理距离和速度。

例如：

```text
1600
```

表示软件按 1600 counts per inch 来换算距离。

注意：

- 输入值必须大于 0。
- 无效输入会回退到最近一次有效值。
- 如果 DPI 输入不准确，报告数、Delta 和间隔仍然可用，但速度和物理距离会不准确。
- 软件只读取并计算数据，不会自动修改鼠标 DPI。

---

### 3.4 实时摘要说明

实时摘要包含当前会话的核心统计。

#### 报告数

报告数表示当前会话中记录到的 Raw Input 鼠标报告数量。

报告数越多，图表和统计越有参考价值。过短的采集段可能无法稳定反映设备表现。

#### X 净位移 / Y 净位移

X 净位移和 Y 净位移分别表示所有 DeltaX、DeltaY 的累加结果。

它们反映从采集开始到当前的净移动方向和距离。来回移动后，净位移可能接近 0，但这不代表鼠标没有移动。

#### 运动路径

运动路径会累计每包位移的路径长度。

它更接近“鼠标实际走过了多长轨迹”，不会像净位移那样被反向移动抵消。

#### 速度

速度基于 Raw Input 位移、时间和输入的 CPI 估算，单位为 m/s。

速度指标适合观察快速甩动、低速微动和不同 DPI 下的实际移动强度。

---

### 3.5 数据质量提示

页面底部的数据质量行可能包含：

- **报告**：采集到的报告数量。
- **零位移**：DeltaX 和 DeltaY 同时为 0 的报告占比。
- **控制**：按键、滚轮等控制类输入占比。
- **丢包**：软件内部缓冲队列溢出丢弃的包数量。
- **质量**：当前数据质量状态，例如良好或降级。
- **队列**：采集队列高水位与容量。

正常情况下，丢包应保持为 0，质量应尽量为良好。

如果质量降级或队列压力较高，可以尝试：

- 缩短采集时长。
- 关闭高负载后台程序。
- 降低鼠标回报率后重新测试。
- 使用更稳定的 USB 接口或无线接收器位置。

---

### 3.6 图表窗口

点击 **图表** 可以打开独立图表窗口。

图表窗口支持：

- 选择图型。
- 切换时间基准。
- 显示或隐藏长上报间隔。
- 选择长上报间隔的分析对象。
- 设置选区起点和终点。
- 开关垂线、连线。
- 导入对比数据。
- 导出当前会话数据。
- 保存 PNG。

图表窗口关闭后，当前页面的采集会话不会被清空。

---

### 3.7 常用图型说明

#### X / Y 轴位移计数-时间

每个点表示一次上报中的 DeltaX 或 DeltaY。

适合观察：

- 单轴噪声。
- 微小抖动。
- 快速移动时是否出现异常大脉冲。
- 横向或纵向移动是否有不自然突刺。

#### 上报间隔-时间

每个点表示当前报告与上一包之间的时间差。

这个图最适合观察主机侧收到报告是否均匀。如果出现明显尖峰，可能来自无线链路、USB、系统调度或采集处理压力。

#### 上报频率-时间

该图把上报间隔转换为局部频率估计，适合观察一段移动中的频率趋势。

它比单个瞬时间隔更平滑，但仍然反映 Raw Input 到达行为。

#### 速度-时间

速度图会结合 CPI 估算 X 轴速度、Y 轴速度、X/Y 速度或路径速度。

如果 CPI 无效，速度图不可用。

#### 上报间隔直方图

直方图显示选区内上报间隔的分布。

标称 1000Hz 的设备，理想间隔约为：

```text
1 ms
```

标称 8000Hz 的设备，理想间隔约为：

```text
0.125 ms
```

实际图形会受到系统计时、调度、设备固件和采集环境影响，不需要期待所有样本都精确落在一个点上。

#### Delta 分布直方图

DeltaX、DeltaY 和 Delta 模长分布用于观察位移输出的量化、偏置和微动比例。

低速微动时，如果大量样本集中在 0 或少数几个计数值上，通常说明当前 DPI、速度或传感器输出量化较明显。

#### X-Y 轨迹

X-Y 轨迹按累计 X 和累计 Y 绘制鼠标路径。

它适合观察：

- 直线是否直。
- 画圆是否圆。
- 是否存在角度修正。
- 是否有蛇形轨迹、偏轴或异常回折。

---

### 3.8 时间基准

图表中有两种时间基准：

#### 原始采集时间

原始采集时间保留真实采集时间轴。

如果中途暂停，暂停造成的时间断档也会保留。

#### 逻辑会话时间

逻辑会话时间会压缩暂停后的断档。

它更适合把多段采集当作连续会话观察，避免暂停时间把图表拉得很长。

---

### 3.9 导入、导出和对比

图表窗口可将当前会话导出为 JSON 文件。

之后可以通过 **导入** 重新加载该文件。

使用方式：

- 当前页面没有实时数据时，导入文件会进入只读浏览模式。
- 当前页面已有实时数据时，导入文件会作为对比鼠标加入图表。
- 图表最多支持 3 个鼠标同图对比。
- 继续导入超过上限时，会提示是否替换最早导入的对比数据。
- **删除导入数据** 可清空导入数据并回到实时待机模式。

导出的 JSON 用于本软件复查和对比，不建议手动修改。

---

### 3.10 如何设计一次可靠测试

如果要比较两只鼠标或两组设置，建议：

1. 使用相同鼠标垫和相同 USB / 接收器环境。
2. 记录并输入正确 DPI。
3. 每次采集做相似动作，例如直线慢移、快速横扫或画圆。
4. 每次采集时长保持接近。
5. 导出基准数据，再导入到另一轮测试中同图对比。
6. 同时观察图表、数据质量和丢包情况。

高回报率设备更容易暴露系统调度和采集压力，因此测试时最好关闭会明显抢占 CPU/GPU 的程序。

---

### 3.11 原理简述

鼠标性能分析会在锁定状态下只记录选中设备的 Raw Input 鼠标数据包。

软件保存每包的时间、DeltaX、DeltaY 和相关控制信息，并基于这些数据构建：

- 实时摘要。
- 运动曲线。
- 上报间隔和频率曲线。
- 速度估算。
- 分布直方图。
- X-Y 轨迹。
- 长间隔覆盖层和统计信息。

需要注意的是，本页时间轴反映的是主机侧 Raw Input 到达行为，不是鼠标内部硬件时钟。

---

## 4. 鼠标灵敏度复制

### 4.1 这个功能是做什么的

鼠标灵敏度复制用于让一只目标鼠标尽量接近另一只源鼠标的手感。

它通过两只鼠标同步滑动的 Raw Input 位移，计算目标鼠标应设置的推荐 DPI 和调整倍率。

本页只做测量和推荐，不会直接修改鼠标驱动或硬件设置。

---

### 4.2 使用前准备

你需要准备：

- 两只在线鼠标。
- 源鼠标当前 DPI。
- 目标鼠标当前 DPI。
- 足够大的鼠标垫或桌面空间。
- 尽量稳定的握持方式。

源鼠标指的是“你想复制手感的鼠标”。

目标鼠标指的是“你想调整到相同手感的鼠标”。

---

### 4.3 基本使用步骤

1. 进入 **鼠标灵敏度复制** 页面。
2. 点击 **开始** 进入绑定与 DPI 设置。
3. 输入 **源鼠标 DPI** 和 **目标鼠标 DPI**。
4. 点击源鼠标区域的 **绑定**，轻微移动源鼠标。
5. 点击目标鼠标区域的 **绑定**，轻微移动目标鼠标。
6. 两只鼠标都绑定成功后，点击 **下一步**。
7. 将两只鼠标并排放好。
8. 点击 **开始第 1 轮**。
9. 同方向、同节奏一起滑动两只鼠标，直到源鼠标和目标鼠标进度都满。
10. 按同样方法完成第 2 轮和第 3 轮。
11. 查看推荐目标设备 DPI 设置和调整倍率。
12. 需要时点击 **复制 DPI**。

每一轮都需要手动开始。切换页面会取消当前未完成轮次，但不会影响已经完成的轮次。

---

### 4.4 绑定说明

绑定时，软件会等待你移动对应鼠标。

注意：

- 轻微移动即可完成绑定。
- 绑定等待时间约为 10 秒。
- 源鼠标和目标鼠标不能是同一只。
- 已绑定鼠标断开后，需要重新绑定。
- 修改 DPI 或重新绑定会清空已测结果。

如果绑定冲突，说明源鼠标和目标鼠标被识别为同一设备；请换另一只鼠标或重新绑定。

---

### 4.5 测量动作建议

测量时建议：

- 两只鼠标并排放置。
- 用同一只手或双手保持同方向、同节奏滑动。
- 尽量滑直线。
- 不要突然加速、急停或画弧线。
- 每轮距离要足够长。
- 每轮不要拖太久。

软件按源 DPI 和目标当前 DPI 计算每轮目标位移量。内部口径相当于每轮约 8 英寸的路径计数目标，因此 DPI 越高，需要累积的计数越多。

---

### 4.6 结果说明

#### 推荐目标设备DPI设置

推荐目标设备DPI设置表示软件建议你把目标鼠标调到的 DPI。

例如：

```text
推荐目标设备DPI设置 1320
```

表示可以尝试把目标鼠标 DPI 设置为 1320。

如果鼠标驱动不支持精确到该数值，可以选择最接近的档位，或使用调整倍率换算游戏内灵敏度。

#### 调整倍率

调整倍率表示目标鼠标当前 DPI 应乘以多少。

例如：

```text
调整倍率 0.825
```

如果目标鼠标当前是 1600 DPI，则推荐值约为：

```text
1600 × 0.825 = 1320
```

也可以把这个倍率用于游戏内灵敏度换算。

#### 稳定性

稳定性表示三轮结果彼此接近的程度。

常见状态包括：

- **极佳**：三轮很接近，可直接参考。
- **良好**：结果已经比较稳。
- **一般**：结果可参考，建议再测一次。
- **较差**：波动偏大，建议重测。

如果三轮差异较大，优先检查滑动动作是否一致、两只鼠标是否同步、桌面空间是否足够。

---

### 4.7 常见失败原因

#### 本轮超时

测量时间太长，通常是移动太慢或中途停顿太久。

处理方法：重新开始本轮，连续完成滑动。

#### 移动过快

本轮完成时间太短，软件认为动作过快，不适合作为稳定测量。

处理方法：放慢一点，保持平稳移动。

#### 位移数据太少

有效 Raw Input 包不足，可能是滑动距离太短、回报率太低或移动太轻。

处理方法：拉长距离，确保两只鼠标都有明显位移。

#### 轨迹过弯

鼠标轨迹不够直，曲率过大。

处理方法：尽量做直线滑动，减少弧线和手腕旋转。

#### 两只鼠标移动角度差距较大

源鼠标和目标鼠标没有朝同一方向移动。

处理方法：重新摆放两只鼠标，让它们同步向同一方向滑动。

#### 两只鼠标的轨迹形态不一致

两只鼠标虽然都有移动，但路径形状差异太大。

处理方法：尽量让两只鼠标以相同节奏、相同路径移动。

#### 两只鼠标不同步

两只鼠标开始、结束或中途速度节奏差异较大。

处理方法：并排放置，同时起步，同时停稳。

---

### 4.8 如何提高结果可靠性

建议：

- 每轮开始前先摆正两只鼠标。
- 用中等速度滑动，不要太快或太慢。
- 保持桌面和鼠标垫平整。
- 尽量减少无线鼠标干扰。
- 如果稳定性只有一般或较差，重新测三轮。
- 如果某一轮明显失败，使用 **重试本轮**。

测得结果后，可以在目标鼠标驱动中设置推荐 DPI，再回到其他页面验证回报率、轨迹和速度表现。

---

### 4.9 原理简述

软件分别记录源鼠标和目标鼠标的 Raw Input 位移。

每轮测量中，它会比较两只鼠标在同一时间窗口内的路径长度、主运动方向、重叠程度和轨迹相似度。

通过这些数据估算：

```text
调整倍率 = 源鼠标实际位移比例 / 目标鼠标实际位移比例
```

最终三轮完成后，软件使用三轮调整倍率的中位数作为最终倍率，并据此计算推荐目标 DPI。

---

## 5. 传感器角度校准

### 5.1 这个功能是做什么的

传感器角度校准用于测量鼠标在你自然横向滑动时，Raw Input 轨迹相对“水平”的偏转角度。

它适合用于：

- 观察鼠标传感器是否存在角度偏移。
- 判断自己握持姿势下的自然水平轨迹。
- 为支持角度校准的鼠标驱动提供参考值。
- 比较不同鼠标、不同握法或不同垫面的轨迹稳定性。

本页只给出推荐角度，不会直接修改设备设置。

---

### 5.2 基本使用步骤

1. 进入 **传感器角度校准** 页面。
2. 点击 **测量区域** 开始，并锁定光标。
3. 用你平时的自然握姿左右来回滑动。
4. 建议完成不少于 **30 次** 往返。
5. 按右键或 `Esc` 暂停。
6. 查看 **推荐角度**、往返数、样本数和稳定度。
7. 点击 **复制角度** 可复制推荐值。
8. 点击 **重置** 可清空当前测量。

暂停后再次点击测量区域，可以在同一会话内继续测量，以提升结果稳定性。

---

### 5.3 滑动建议

测量时建议：

- 使用日常握姿，不要刻意把鼠标摆得很正。
- 左右来回滑动，尽量维持你主观认为的水平。
- 每次滑动保持直线。
- 两侧幅度尽量均衡。
- 避免急转折、抖动和突然加速。
- 如果结果不稳定，继续补充更多往返。

这个功能测的是“你当前握姿和设备组合下的实际轨迹偏角”，因此刻意纠正握姿可能会让结果失去参考意义。

---

### 5.4 指标说明

#### 推荐角度

推荐角度是软件根据左右滑动轨迹拟合出来的角度。

只有完成足够往返并且轨迹质量可用时，推荐角度才会显示。

正负号表示需要参考软件显示的方向约定。使用到鼠标驱动或其他软件中时，如果发现校正方向相反，可以尝试取相反数验证。

#### 往返

往返表示软件接受到的有效左右滑动次数。

建议至少达到 30 次后再参考推荐角度。

#### 样本

样本表示参与测量的有效水平运动片段数量。

样本越多，通常结果越稳。但低质量样本过多也可能拉低质量评分。

#### 稳定

稳定表示最近推荐角度候选值的波动，单位是角度。

简单理解：

- 数值越小，结果越稳定。
- 数值越大，说明最近几次轨迹拟合差异较大。

---

### 5.5 轨迹质量提示

页面会根据当前数据给出质量提示。

常见提示包括：

- **轨迹样本仍偏少**：继续完成更多往返。
- **左右幅度还不够均衡**：让左右两侧摆动更对称。
- **轨迹离散较大**：尽量保持轨迹更直、更稳定。
- **异常轨迹较多**：减少抖动和急转折。
- **采集队列曾溢出**：系统压力可能影响结果，建议重置后重测。
- **轨迹质量良好 / 优秀**：可以继续保持当前节奏确认结果。

如果质量长期较差，不建议直接采用推荐角度。

---

### 5.6 如何使用推荐角度

如果你的鼠标驱动支持传感器角度校准，可以把推荐角度填入对应设置项。

使用时建议：

1. 先记录原始设置。
2. 输入推荐角度。
3. 在游戏或绘图软件中做直线测试。
4. 如果方向变得更歪，尝试使用相反符号。
5. 如果不同轮次差异较大，回到本页重置后再测一次。

不要只凭一次短测就长期固定角度。最好做 2-3 次独立测量，确认推荐值接近后再采用。

---

### 5.7 原理简述

软件会记录鼠标左右滑动时的 Raw Input 位移，并把轨迹分成有效的左右滑动段。

每段轨迹需要满足一定长度、水平占比、持续时间、直线度和点数要求，才会参与角度拟合。

软件分别统计向左和向右滑动的角度样本，剔除明显离群值后，计算左右两侧的综合角度，再生成推荐角度。

完成足够往返后，推荐角度才会显示。稳定度来自最近一段时间内候选角度的波动。

## 7. 测试结果的通用注意事项

### 7.1 Raw Input 结果代表什么

ClickSyncMouseTester 观察的是 Windows Raw Input 层收到的数据。

它能很好地反映当前电脑、系统、驱动、USB / 无线链路和应用处理环境下的实际输入表现。

但它不是硬件总线分析仪，也不能直接读取鼠标内部传感器时钟。

因此，结果应理解为：

> 当前 Windows 用户态环境中，应用实际能收到的输入表现。

---

### 7.2 影响测试结果的因素

常见影响因素包括：

- 鼠标固件和驱动设置。
- 回报率、DPI、节能模式。
- USB 接口、扩展坞、集线器。
- 无线接收器距离和干扰。
- 系统负载、后台程序、录屏软件。
- 显示器刷新率和桌面合成负载。
- 鼠标垫表面和移动方式。

如果结果异常，建议先固定其他条件，只改变一个变量重新测试。

---

### 7.3 建议的测试习惯

为了得到更可靠的结果：

- 每次测试前确认选中的设备正确。
- 同一对比实验使用相同 DPI、回报率和移动方式。
- 高回报率测试时关闭不必要后台程序。
- 记录鼠标型号、接口、驱动设置和测试环境。
- 不要只看单次瞬时值，结合峰值、分布、图表和质量提示判断。
- 出现丢包或质量降级时，优先排查系统负载和接口环境。

这样得到的结果更适合横向对比，也更方便复现问题。
